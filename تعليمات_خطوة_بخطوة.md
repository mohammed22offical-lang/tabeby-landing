  `c:\Users\AEEN.IQ\Desktop\tabeby landing`

## الجزء ١ — تشغيل المشروع على جهازك

### الخطوة ١
**المطلوب:** تثبيت المكتبات (مرة واحدة).

**اعمل كذا:**
```powershell
npm install
```

**المفروض تشوف:** سطر أو أكثر يكتب "added XXX packages" وما يطلع خطأ أحمر.

**تابع تقدمك:** □ ثبّت المكتبات (npm install)

---

### الخطوة ٢
**المطلوب:** تشغيل موقع التطوير على جهازك.

**اعمل كذا:**
```powershell
npm run dev
```

**المفروض تشوف:** رسالة مثل "Local: http://localhost:3000".

**اعمل كذا بعدها:** افتح المتصفح واكتب في شريط العنوان:
```
http://localhost:3000
```

**تابع تقدمك:** □ شغّلت الموقع (npm run dev) وفتحته في المتصفح

---

### الخطوة ٣
**المطلوب:** إيقاف السيرفر.

**اعمل كذا:** في نفس نافذة الطرفية اللي فيها `npm run dev` اضغط:
```
Ctrl + C
```

**تابع تقدمك:** □ عرفت كيف أوقف السيرفر (Ctrl+C)

---

## الجزء ٢ — بناء المشروع للإنتاج

### الخطوة ٤
**المطلوب:** بناء المشروع (استعداداً للنشر أو التسليم).

**اعمل كذا:**
```powershell
npm run build
```

**المفروض تشوف:** في الآخر "Compiled successfully" و "Generating static pages" بدون أخطاء حمراء.

**تابع تقدمك:** □ البناء نجح (npm run build)

---

### الخطوة ٥
**المطلوب:** تشغيل النسخة المُبناة (مثل السيرفر الحقيقي).

**اعمل كذا:**
```powershell
npm start
```

**اعمل كذا بعدها:** افتح المتصفح على:
```
http://localhost:3000
```

**تابع تقدمك:** □ شغّلت النسخة النهائية (npm start)

---

### الخطوة ٦
**المطلوب:** إيقاف السيرفر مرة ثانية.

**اعمل كذا:** في نافذة الطرفية اضغط:
```
Ctrl + C
```

**تابع تقدمك:** □ أوقفت السيرفر

## الجزء ٣ — رفع المشروع على GitHub

### الخطوة ٧
**المطلوب:** التأكد إنك داخل مجلد المشروع.

**اعمل كذا:**
```powershell
cd "c:\Users\AEEN.IQ\Desktop\tabeby landing"
```

**تابع تقدمك:** □ أنا داخل مجلد المشروع

---

### الخطوة ٨
**المطلوب:** إنشاء مستودع Git والفرع الرئيسي `main`.

**اعمل كذا:**
```powershell
git init -b main
```

**المفروض تشوف:** "Initialized empty Git repository..."

**تابع تقدمك:** □ أنشأت المستودع (git init -b main)

---

### الخطوة ٩
**المطلوب:** إضافة كل الملفات الجاهزة للرفع (بدون node_modules و .next).

**اعمل كذا:**
```powershell
git add .
git status
```

**المفروض تشوف:** قائمة ملفات (مثل src، package.json، README...) **بدون** مجلد node_modules وبدون .next.

**تابع تقدمك:** □ أضفت الملفات (git add .) وتأكدت من القائمة (git status)

---

### الخطوة ١٠
**المطلوب:** عمل أول حفظ (commit).

**اعمل كذا:**
```powershell
git commit -m "Initial commit: Tabeby landing"
```

**المفروض تشوف:** "X files changed" و "create mode..." بدون أخطاء.

**تابع تقدمك:** □ عملت أول commit

---

### الخطوة ١١
**المطلوب:** إنشاء المستودع على GitHub وربطه بمشروعك.

**اعمل كذا:**
1. ادخل على [github.com](https://github.com) وسجّل دخولك.
2. اضغط **New** (أو New repository).
3. سمّ المستودع (مثلاً: `tabeby-landing`).
4. **لا** تضف README ولا .gitignore ولا رخصة (اترك المستودع فاضي).
5. اضغط **Create repository**.
6. انسخ رابط المستودع (HTTPS)، مثلاً:  
   `https://github.com/اسمك/tabeby-landing.git`

**اعمل كذا في الطرفية** (غيّر الرابط لرابطك):
```powershell
git remote add origin https://github.com/اسمك/tabeby-landing.git
```

**تابع تقدمك:** □ أنشأت المستودع على GitHub وربطته (git remote add origin)

---

### الخطوة ١٢
**المطلوب:** رفع المشروع على GitHub.

**اعمل كذا:**
```powershell
git push -u origin main
```

**المفروض تشوف:** "Enumerating objects..." ثم "Writing objects: 100%" ثم "main -> main".

**إذا طلع خطأ:** راجع ملف **GITHUB_SETUP.md** في المشروع، فيه حلول لأخطاء الرفع (مثل push rejected أو ملف كبير).

**تابع تقدمك:** □ رفعت المشروع (git push -u origin main)

---

## خلاص — ماذا بعد؟

- **لتسليم المشروع لمبرمج:** أرسل له رابط المستودع على GitHub. هو يشغّل `npm install` ثم `npm run build` و `npm start` أو ينشره على Vercel/Netlify.
- **للتطوير لاحقاً:** كل مرة تغيّر فيها ملفات:
  ```powershell
  git add .
  git commit -m "وصف التعديل"
  git push
  ```

## ملخص سريع (للمراجعة)

| ماذا تريد؟        | الأمر |
|-------------------|--------|
| تثبيت المكتبات    | `npm install` |
| تشغيل التطوير     | `npm run dev` |
| بناء للإنتاج      | `npm run build` |
| تشغيل النسخة النهائية | `npm start` |
| إيقاف السيرفر     | `Ctrl + C` |
| أول رفع لـ GitHub | `git init -b main` → `git add .` → `git commit -m "..."` → `git remote add origin ...` → `git push -u origin main` |
